## PART 2 — The 8-Stage Sponsorship Sales Process

From SponsorFlo's Sponsorship Sales Guide (updated October 2024):

### Stage 1: Prospecting & Research
- Build ideal sponsor profile (industry, budget, audience fit, past sponsorships)
- Research company marketing objectives
- Identify decision-makers (marketing director, partnership manager, CMO)
- Find warm introduction paths (LinkedIn mutual connections, past event attendees)

**mdeai implementation:**
- `sponsor-audience-match` fn: brand description → matching events
- Sinapser-style: scrape competitor event sponsor pages → score each brand
- Hermes Python: score 50 prospects in batch using the 5-factor formula
- DB: `sponsor.discovery_pipeline` table tracking source, score, status

### Stage 2: Initial Outreach
- Personalized email campaigns
- LinkedIn strategic connections
- Phone follow-ups with value
- Reference mutual connections

**mdeai implementation:**
- OpenClaw: WhatsApp first (Colombia-specific); email second
- `sponsor-creative-gen`: generate personalized ES+EN outreach copy
- Cadence: Day 1 WA intro → Day 4 follow-up → Day 8 final with social proof
- Track reply rates per channel in `sponsor.outreach_log` table

### Stage 3: Discovery & Needs Analysis
- Ask about marketing objectives
- Identify target demographics
- Understand budget parameters
- Map decision-making process

**mdeai implementation:**
- `sponsor_concierge` chat agent qualification sequence:
  - Q1: Budget (→ tier routing)
  - Q2: Industry (→ event category match)
  - Q3: Goal (awareness / leads / sampling / engagement)
- Results stored to `sponsor.applications.campaign_goals` JSONB

### Stage 4: Proposal Development
- Tailor packages to objectives
- Include data-driven projections
- Show competitive advantages
- Provide clear ROI metrics

**mdeai implementation:**
- `sponsor-proposal-gen` fn: Pro model, generates bilingual PDF content
- Include: event audience demographics, past CTR from `sponsor.roi_daily`, comparable deals
- `sponsor-price-recommend` fn: "Events like yours charge $X–$Y for Silver"
- Template: `sponsor.contracts.agreed_deliverables` JSONB as deliverable checklist

### Stage 5: Presentation & Pitch
- Tell engaging stories
- Use visuals and data
- Demonstrate past success
- Create urgency naturally

**mdeai implementation:**
- AI mockup equivalent: Supabase Storage preview of how brand logo will appear on SponsoredSurface
- Show live impression numbers from existing sponsors (social proof)
- `/sponsor/apply` wizard as self-serve pitch-to-close flow

### Stage 6: Objection Handling
- Listen and acknowledge concerns
- Provide evidence and examples
- Offer flexible solutions
- Reframe value proposition

**mdeai implementation:**
- `sponsor_concierge` chat agent: handles "too expensive" → shows Bronze tier
- Admin can adjust package via `/admin/sponsorships/:id`
- Flexible: allow custom package beyond standard tiers

### Stage 7: Negotiation & Closing
- Start with anchor pricing (highest tier first)
- Bundle for perceived value
- Use trial closes
- Create win-win outcomes

**mdeai implementation:**
- Default tier shown first: Gold (anchor), then offer Silver
- "First event free" (0% commission) as closing offer
- Stripe checkout as friction-free close

### Stage 8: Onboarding & Activation
- Welcome and orientation
- Set clear expectations
- Quick wins in first 30 days

**mdeai implementation:**
- Contract sign → placements activated automatically
- Sponsor dashboard live on day 1 (task 052)
- First AI insight delivered within 24h (sponsor-roi-explain cron)
- WA welcome message via OpenClaw with dashboard link + account manager contact

## PART 3 — 6 additional platforms discovered

### Platform A: Sinapser (`sponsors.sinapser.com`)

**What it does:** Semantic vector matching of events to historical sponsors.
- 2,700+ verified sponsors in DB
- 6,000+ historical sponsorships analyzed
- AI generates semantic profile of your event → cross-references against real sponsorship records using vector embeddings
- Each match includes: AI explanation of why they match, similar events they've sponsored, decision-maker contact (name, title, email, LinkedIn)

**Business model:** Free to search. Pay-per-use credits for contact discovery, sponsor expansion, personalized emails. No subscriptions.

**mdeai equivalent:** `sponsor-audience-match` fn (Pro + googleSearch) + Firecrawl scraping of competitor event sponsor pages → vector similarity search in `sponsor.marketplace_index.embedding`

**Key feature to copy:** "AI-generated explanation of why this brand matches your event" — include this in the audience match output schema.

### Platform D: ShowCare — AEO for Events

**Core thesis:** Professionals are using ChatGPT, Perplexity, and Google AI Mode to find events. If AI can't find your event, it can't recommend it. **AEO (Answer Engine Optimization) converts 4.4x better than traditional organic search visitors.**

**8 practical recommendations:**
1. FAQ with specific, citable facts (not vague marketing copy)
2. Replace "thousands of attendees" with "14,200 supply chain professionals, 82% implemented a strategy within 90 days"
3. Speaker + session content: include specific learning outcomes
4. Add Schema.org event schema markup to all event pages
5. Update year-round (not just event week)
6. Build credibility through third-party validation
7. Turn event content into year-round AI fuel
8. Track AI visibility

**mdeai action:** Add Schema.org `Event` markup to every event detail page (`/events/:id`) and every marketplace event profile (`/marketplace/events/:eventId`). This makes mdeai-hosted events discoverable via ChatGPT/Perplexity queries like "fashion events in Medellín Colombia."

### Platform E: Vendelux / Event Tech Live — Retention Risk Matrix

**Core insight from Sasha Bartashnik (Head of Data and AI, Vendelux):** "Most organizers have more data than they can interpret." The solution is the **retention risk matrix**.

**Retention risk matrix:**
Plot sponsors on two axes:
- X axis: Execution quality (how well they ran their presence at the event)
- Y axis: ROI potential (how much business the audience could realistically generate for them)

This creates 4 actionable quadrants:
1. **High execution + High ROI potential** → Renew immediately, offer multi-year deal
2. **High execution + Low ROI potential** → Reframe: find new activation approach that unlocks their potential
3. **Low execution + High ROI potential** → Invest in their success: activation support, better placement
4. **Low execution + Low ROI potential** → Don't renew; reallocate to better-fit brands

**mdeai implementation:** Add this 2×2 matrix to the sponsor dashboard as an admin view. Calculate:
- Execution = fulfillment rate (deliverables completed / total agreed)
- ROI potential = `sponsor.roi_daily` CTR vs category benchmark

## PART 4 — Industry pricing benchmarks (from SponsorFlo research)

### CPM rates by asset type

| Asset type | CPM range |
|---|---|
| Live event signage | $15–50 |
| Digital / social assets | $5–25 |
| Premium placements (naming rights) | >$100 |
| Social posts | $500–5,000 per post (by follower count + engagement) |
| Email placements | $0.05–0.50 per subscriber per send |
| Website banners | $5–25 (standard digital CPM) |

### Package structure best practices

- Title sponsorship: **25–40% of total sponsorship revenue target**
  - Example: if total target is $500K, title sponsor = $125K–200K
- Exclusivity premium: **+20–50% to base price** (category exclusivity)
- Multi-year discounts: Year 1 full price / Year 2 5–10% off / Year 3+ 10–15% off
- Annual escalation clause: +3–5% per year
- Recommended tiers: **3 tiers minimum** (most common) up to 5

### ROI benchmarks sponsors expect

- Minimum acceptable ROI: **3:1** ($3 value for every $1 spent)
- High-performing sponsorships: 5:1 to 10:1
- ROI above 200% = strong
- AI-enhanced tracking: 2.5x brand awareness, 5x lead generation, 3.2x customer LTV multipliers

## PART 8 — Key insights that change the strategy

### 1. SponsorFlo uses Claude Sonnet 4 for contract extraction

This confirms mdeai should use **Gemini Pro with urlContext** (equivalent capability) for the `sponsor-contract-extract` fn. No need to add Anthropic API as a dependency — Gemini Pro's document understanding is comparable.

### 2. Sinapser uses vector embeddings against historical sponsorship records

This is the most technically elegant discovery mechanism found. mdeai can implement this natively:
- Store real historical deal data in `sponsor.roi_daily` + `sponsor.applications`
- Embed each event profile using `text-embedding-004`
- New brand: "describe your target audience" → embed → cosine similarity → top 5 matching events
- mdeai has the data advantage because it owns real transaction data, not just self-reported profiles

### 3. The 8-stage process reveals the biggest gap: Stage 3 (Discovery)

Every platform is weak at Stage 3 (needs analysis). SponsorFlo's `sponsor_concierge` chat sequence (in task 059) is actually the most sophisticated implementation of Stage 3 in the market because:
- It happens in WhatsApp/chat (native to Colombia)
- It's available 24/7 (no sales rep needed)
- It routes to the right tier automatically

This is a genuine competitive advantage over every platform in the comparison.

### 4. AEO (Answer Engine Optimization) is an immediate 2-hour win

Adding Schema.org Event markup to all event pages costs nothing to implement and makes every mdeai-hosted event discoverable via ChatGPT/Perplexity. ShowCare's research (4.4x conversion) suggests this is higher ROI than most features on this list.

### 5. SponsorFlo's payment fee structure reveals mdeai's pricing advantage

SponsorFlo charges: Stripe 2.9% + platform fee 1.0-2.5% + $299-799/mo subscription.

mdeai model: Stripe 2.9% + 15% commission on deal value (no subscription, no setup, no minimum).

For a $2,000 Bronze deal:
- SponsorFlo: $57.80 Stripe + $20-50 platform fee + $299-799/mo subscription = **much higher total cost**
- mdeai: $57.80 Stripe + $300 commission = **only 15% on deal value when you win**

The "you only pay when you get paid" model wins for small organizers. This is the Sponsoo model applied to Colombia.
