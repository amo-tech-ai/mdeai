# C09 — CRM Pipeline + Sponsor, Buyer, Seller Chat Flows

## Overview

Three user types with zero chat support today:
1. **Sponsors** — want to sponsor events; currently must navigate to /sponsor/apply
2. **Buyers** — want to purchase property; no investment analysis, no project search
3. **Sellers** — want to sell; no valuation, no lead matching

Plus: an admin CRM view so ops can see and manage all leads generated by chat.

## Part A — Sponsor Intake via Chat

### A1. `sponsor_intake` tool

```typescript
{
  name: "sponsor_intake",
  description: "Help a company or individual apply to sponsor an event on mdeai. Use when user mentions sponsoring, brand visibility, event partnership, or marketing budget.",
  parameters: {
    company_name: { type: "string" },
    contact_name: { type: "string" },
    budget_usd: { type: "number" },
    event_interest: { type: "string", description: "What type of events interest them" },
    goals: { type: "string" }
  }
}
```

Calls `sponsor-application-create` edge function (already exists). Returns OPEN_SPONSOR_APPLIED action.

5-step conversation:
```
User: I want to sponsor events on mdeai
AI: Excellent! We have great sponsorship packages for Medellín events. 
    What company are you representing?

User: Bancolombia
AI: Bancolombia — perfect fit for our events audience. 
    What's your approximate sponsorship budget per event?

User: $2,000–$5,000
AI: Great range. What kind of events interest you most? 
    (Music, business, cultural, nightlife, sports)

User: Music and cultural events
AI: Got it. And what are your key goals? 
    (Brand awareness, lead generation, employee engagement, community presence)

User: Brand awareness and lead generation
AI: Creating your sponsor application now...
    [Application submitted to mdeai sponsorship team]
    [OPEN_SPONSOR_APPLIED action]
    "View your application →" button
```

### A2. Router intents

```typescript
sponsor_apply: { agent: 'sponsor', description: 'Company or individual wants to sponsor events' },
sponsor_inquiry: { agent: 'sponsor', description: 'Questions about sponsorship packages and ROI' },
```

Patterns:
```typescript
{ patterns: [/\b(sponsor|sponsorship|brand.*event|advertise.*event|partnership)\b/i], intent: 'sponsor_apply', confidence: 0.90 },
```

## Part B — Buyer / Investment Chat Flows

### B1. `search_projects` tool

New construction projects and resale properties for buyers:
```typescript
{
  name: "search_projects",
  description: "Search for real estate investment opportunities, new construction projects, and resale properties in Medellín for buyers.",
  parameters: {
    neighborhood: { type: "string" },
    property_type: { type: "string", enum: ["apartment", "house", "commercial", "land"] },
    budget_min: { type: "number" },
    budget_max: { type: "number" },
    bedrooms: { type: "integer" },
    purpose: { type: "string", enum: ["live", "rent_out", "flip"] }
  }
}
```

Data source: `properties` table (needs to be created or extended from `apartments` with `listing_type = 'sale'`).

### B2. `investment_analysis` tool

```typescript
{
  name: "investment_analysis",
  description: "Calculate ROI, cap rate, and cash flow for a property investment in Medellín.",
  parameters: {
    purchase_price_usd: { type: "number" },
    monthly_rent_usd: { type: "number" },
    neighborhood: { type: "string" },
    property_type: { type: "string" }
  }
}
```

Implementation (no external API needed — use Gemini to calculate):
```typescript
const analysis = {
  annual_rental_income: monthly_rent * 12,
  annual_expenses: monthly_rent * 12 * 0.20,  // 20% vacancy/maintenance estimate
  net_operating_income: annual_rental_income - annual_expenses,
  cap_rate: (net_operating_income / purchase_price) * 100,
  cash_on_cash_return: cap_rate * 0.85,  // rough estimate with typical Colombia financing
  payback_years: purchase_price / net_operating_income,
  medellin_market_note: "Medellín cap rates typically 6-9% for Poblado/Laureles apartments (2025 data)"
};
```

Returns structured analysis + renders as an AnalysisCard in chat.

### B3. BuyerAgent system prompt

## Part C — Admin CRM Dashboard

### C1. /admin/leads page

New admin page showing all chat-generated leads:

```tsx
// Table columns
// Lead ID | Intent | Neighborhood | Budget | Email | Status | Created | Agent
// [Assign] [Contact] [Mark Qualified] [Mark Lost]
```

Wire to `leads` table with RLS for admin/service role.

### C2. Lead scoring with Hermes

Hermes ranking can score leads by intent strength:
- 3+ turns on same topic: +20
- Provided email: +30
- Provided phone: +40
- Specific budget mentioned: +20
- Move-in date mentioned: +20
- "I want to apply" or "schedule showing": +50

Score 80+ → "Hot lead" → immediate WA notification to agent.

### C3. CRM pipeline stages

```
new → contacted → showing_scheduled → applied → closed_won / closed_lost
```

Each stage transition:
- new → contacted: automated WA outreach
- contacted → showing_scheduled: add to calendar (future)
- showing_scheduled → applied: send application link
- applied → closed: flag for commission
