Best Practices

Use PostgresStore with explicit id, connectionString, and controlled SSL config. Official PostgreSQL storage docs list ssl, schemaName, pool reuse, indexes, and disableInit for CI/CD:

PostgreSQL storage

.

For production, prefer Supabase pooler when IPv6/direct DB is unreliable; verify the same URL with plain pg/psql before blaming Mastra.

Do not set NODE_TLS_REJECT_UNAUTHORIZED=0 in production. If forced locally, scope it to local smoke only.

Register every visible primitive in src/mastra/index.ts: agents, workflows, scorers, storage, editor.

For memory, always pass both thread and resource in .generate()/.stream() calls. Official memory docs say both are required for storage-backed memory.

Use mastra.getAgentById() / mastra.getWorkflow() inside runtime code so shared storage/logger/registry are applied.

Use workflows for deterministic rental/event paths; use agents for open-ended reasoning.

Keep Mastra as AI runtime, not source of truth. Supabase remains the mdeAI ledger.
