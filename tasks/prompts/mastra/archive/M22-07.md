## 1. Purpose

`tasks/prompts/mastra/` (PRD, dashboard, individual task plans) is referenced from PR #22's body and from [`.claude/skills/mastra/links.md`](../../../../.claude/skills/mastra/links.md), but the directory is **untracked** in `/home/sk/mde`. Anyone reviewing the PR cannot see the planning artifacts the implementation claims to satisfy.

## 2. Goals

- `tasks/prompts/mastra/**` is committed and tracked.
- `tasks/prompts/mastra/dashboard.md` reflects which tasks are Done vs In Progress vs Not Started.

## 3. Features

- Reviewers can open the linked plan in `links.md` and see real content.

## 4. Workflow

1. `git add tasks/prompts/mastra/` from `/home/sk/mde` (canonical checkout).
2. Confirm no secrets or env files are inside that directory before staging.
3. Add a one-line entry to `dashboard.md` reflecting current task statuses.
4. Commit on its own focused branch (per [`mde-worktree-pr-flow`](../../../../.claude/skills/mde-worktree-pr-flow/)).

## 5. User journeys

- **PR reviewer** — clicks the "Mastra task PRD" link in `links.md` and sees the actual file, not a 404.

## 12. Testing Strategy

**Confirm the gap:**
```bash
git ls-files tasks/prompts/mastra/ 2>/dev/null | wc -l
# Expected: 0  ← the directory is untracked

git status --short tasks/prompts/mastra/
# Expected: ?? tasks/prompts/mastra/
```

**After staging:**
```bash
git add tasks/prompts/mastra/
git diff --staged --name-only | head -20
# Expected: lists all M22-*.md, README.md, notes/*.md, etc.

git diff --staged | grep -Ei "KEY|SECRET|TOKEN"
# Expected: no output (no secrets)

git ls-files --cached tasks/prompts/mastra/ | wc -l
# Expected: ≥ 12 files (9 M22 tasks + README + notes + dashboard)
```

**Verify dashboard.md has correct statuses:**
```bash
cat tasks/prompts/mastra/fix/README.md | grep -E "M22-0[1-9]"
# Verify M22-01 shows "In Progress", others show "Not Started"
```
