## 1. Purpose

`my-mastra-app/src/mastra/tools/search-attractions.ts` already supports `freeOnly: boolean` as both an input field (line 142, 174) and a filter (line 157). But `concierge-routing-workflow.ts` does not parse "free" / "no cost" / "gratis" from the user message, does not include `freeOnly` in its dispatch payload, and does not surface free-attraction shortcuts in the prompt. Free-attraction queries silently fall through to a generic search.

## 2. Goals

- Classifier detects free-intent phrasing and sets `freeOnly: true` in the dispatch payload.
- Workflow's attraction branch forwards `freeOnly` to `searchAttractionsTool`.
- "Free museums in Centro" returns only entries with `entryFee: 0`.

## 3. Features

- "Free attractions in Comuna 13", "anything free this weekend", "no-cost museums" all return free-only results.

## 4. Workflow

1. Add a regex/heuristic to the classifier in `concierge-routing-workflow.ts`:
   - matches: `\b(free|gratis|no.cost|complimentary)\b` → set `freeOnly: true`.
2. Extend the attraction-dispatch payload schema with `freeOnly: z.boolean().optional()`.
3. Forward `freeOnly` into `searchAttractions({ ... freeOnly })`.
4. Add a regression test in `M22-06` for `"free museums in Centro"` → classifier output includes `freeOnly: true` + intent `attraction_search`.

## 5. User journeys

- **Andrés (budget traveler)** — "free attractions today" returns Plaza Botero, Comuna 13 graffiti tour, etc., not pay-to-enter venues.

## 6. Agents

| Agent | Edge fn | Model | What it does |
|---|---|---|---|
| router/classifier inside concierge-routing-workflow | n/a | deterministic regex first, model fallback `openai/gpt-5.4-mini` | Detects "free" phrasing, sets `freeOnly` |

## 7. Integrations

| Integration | Purpose | Auth source |
|---|---|---|
| `searchAttractionsTool` | Filter mock attraction catalog by `freeOnly` | n/a (in-process) |

Recognize "free" phrasing in the classifier and forward `freeOnly` into the attraction tool. The tool already filters; the workflow just wasn't passing the flag.
