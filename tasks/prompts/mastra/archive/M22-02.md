# M22-02 — Add timeouts to weather-tool fetch calls

> **Summary:** Two `fetch()` calls in `weather-tool.ts` have no timeout. A slow Open-Meteo API silently hangs the agent for 30+ seconds. Fix: `AbortSignal.timeout(5000)` on both calls, wrap in try/catch to return a clean error.
>
> **Verification:** Bug confirmed on `origin/main` — `grep -n "fetch(" my-mastra-app/src/mastra/tools/weather-tool.ts` shows bare fetches at lines 45 and 56 with no `signal:` option.
>
> **Status:** 🟢 Not Started · Unblocked · Start now

## 1. Purpose

`my-mastra-app/src/mastra/tools/weather-tool.ts` makes two unbounded `fetch()` calls (lines 45 and 59) to `geocoding-api.open-meteo.com` and `api.open-meteo.com`. With no timeout, a degraded upstream silently hangs the agent and any workflow that calls it. Mastra workflows get marked as `failed` with `ETIMEDOUT` from a network layer minutes later, instead of a clean tool-level failure in seconds.

## 2. Goals

- Both fetches abort after **5 seconds**.
- On timeout the tool returns a structured error, not an unhandled rejection.
- A workflow that calls the weather tool with a deliberately blocked DNS still completes within 10 s with `status: 'failed'` and a readable error.

## 3. Features

- Reliable smoke runs: weather-workflow no longer needs the "external-network-blocked" exception in [`my-mastra-app/scripts/mastra-smoke.sh`](../../../../my-mastra-app/scripts/mastra-smoke.sh).
- Faster failure surfacing in Studio traces.

## 4. Workflow

1. Replace `await fetch(geocodingUrl)` with `await fetch(geocodingUrl, { signal: AbortSignal.timeout(5000) })`.
2. Same for `await fetch(weatherUrl, ...)`.
3. Wrap both in `try/catch`; on `AbortError` or network error, throw a `Error('weather upstream unavailable')` so the tool's caller can render an empty state.
4. Smoke-test by setting `MASTRA_DEV_NO_CACHE=1` and pointing DNS at an unreachable host (or `curl -m 1` the same URLs first to confirm the path).

## 5. User journeys

- **Camila uses concierge for a Medellín weather follow-up** — if Open-Meteo is down she gets "Weather data unavailable, here's what we know about Medellín…" within 5 s instead of a 30 s spinner.

## 6. Agents

None directly — but `weather-agent` and any workflow consuming `weather-tool` benefits.

## 7. Integrations

| Integration | Purpose | Auth source |
|---|---|---|
| Open-Meteo geocoding | city → coords | none (public) |
| Open-Meteo forecast | coords → temp | none (public) |

## 8. Summary

Cap the two outbound fetches at 5 s with `AbortSignal.timeout()`. Convert hangs into clean failures. No new dependency, no new abstraction.
