## 7. Best-Practice Improvements

- **Smoke:** Prefer a **temporary env file assembled from `(process ∪ .env)`** when invoking `mastra dev --env`, instead of blindly pointing at `.env` when Postgres fallback is inactive (aligns with setup expectations in **[mastra-smoke-test](../../../../.claude/skills/mastra-smoke-test/SKILL.md)**).
- **`fetch`:** **`AbortSignal.timeout(ms)`** (or AbortController + timer) on **every** outbound HTTP from tools (**[mastra links — workflows/errors patterns](https://mastra.ai/docs/workflows/error-handling)**).
- **Memory schema:** Extend concierge working memory enums / fields for **`restaurant_search`** and **`attraction_search`** (**[Memory overview](https://mastra.ai/docs/memory/overview)**).
- **`freeOnly`:** Thread through classifier → dispatcher → **`searchAttractions`** input.
- **Tests:** Minimal **deterministic classifier** regressions (`1BR Laureles`, `free museums`, `cheap salsa`, `coffee Poblado`).
- **Lint order:** Lint **clean source** paths; Mastra **`mastra build` after lint** CI ordering if codegen pollutes (**this repo ignores `**/.mastra/**`** at root ESLint layer when config present).
- **Git hygiene:** **`git add tasks/prompts/mastra`** for notes/prompts you treat as authoritative.
