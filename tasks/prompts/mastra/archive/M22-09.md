# M22-09 — Add `bedrooms` to cardSchema; reranker reads typed field, not regex on subline

> **Summary:** The rental reranker parses bedrooms from the human-readable `subline` display string via regex (`card.subline.match(/\d+BR/)`). This breaks silently when display copy changes. The raw rental record has `bedrooms: number` — it just isn't mapped into `cardSchema`. Fix: add `bedrooms: z.number()` to `cardSchema`, populate it in the mapping step, and have the reranker read `card.bedrooms` directly.

## 1. Purpose

`my-mastra-app/src/mastra/workflows/rental-search-workflow.ts` line 196 reranks candidates by parsing bedrooms out of the human-readable `subline` string with `card.subline.match(/\d+BR/)`. The string is a display field meant for the user. If the subline format changes (e.g. localized to "1 alcoba"), the reranker silently breaks. The rental record itself already has a typed `bedrooms: number` (line 11), but `cardSchema` drops it.

## 2. Goals

- `cardSchema` includes `bedrooms: z.number()`.
- Reranker reads `card.bedrooms`, not `card.subline.match(...)`.
- Subline format can change without breaking ranking.

## 3. Features

- Stable ranking when card display strings change.

## 4. Workflow

1. Add `bedrooms: z.number()` to `cardSchema` in `rental-search-workflow.ts` (around line 36–40).
2. Populate `bedrooms: r.bedrooms` in the mapping step (around line 87).
3. Replace the regex parse at line 196 with `card.bedrooms >= 2` (or whatever threshold the rerank uses).
4. Search for any other consumer that parsed bedrooms from `subline` and update.
5. Cover with a unit test: card with `bedrooms: 2` ranks above `bedrooms: 0` for a "≥2 bedrooms" query.

## 5. User journeys

- **Camila with kids** asks for ≥2-bedroom rental — gets multi-bedroom options at the top regardless of subline phrasing.

## 6. Agents

None — pure workflow change.

## 7. Integrations

None.

## 8. Summary

Stop regex-parsing display strings. Add the typed field to the card schema and reranker reads it directly. Removes silent breakage when display copy changes.
