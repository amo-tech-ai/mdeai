## 1. Purpose

`concierge-routing-workflow.ts` ships a deterministic regex/heuristic classifier (`classifyDeterministic()`). Two real bugs already shipped from this function: the rentalSignals regex didn't match `1BR`, and `pickPrice` grabbed `1` from `1BR` and `13` from `Comuna 13`. Both were caught by hand-running Studio. There are zero unit tests guarding the regex set, so the next bug ships the same way.

## 2. Goals

- Vitest spec for `classifyDeterministic()` in `my-mastra-app/src/mastra/workflows/__tests__/concierge-routing.test.ts`.
- Coverage for every shipped regression + every intent class.
- `npm --prefix my-mastra-app run test` runs and passes; package.json no longer says `"Error: no test specified"`.

## 3. Features

- CI catches classifier regressions before they reach Studio.

## 4. Workflow

1. Add `vitest` to `my-mastra-app/devDependencies`.
2. Replace the `test` script in `my-mastra-app/package.json` with `vitest run`.
3. Write specs covering at least:
   - `"1BR Laureles under $80"` → `intent: 'rental_search'`, `bedrooms: 1`, `neighborhood: 'Laureles'`, `maxPricePerNight: 80`.
   - `"Comuna 13 graffiti tour"` → `intent: 'attraction_search'`, neighborhood: 'Comuna 13', no spurious `price=13`.
   - `"free museums in Centro"` → `intent: 'attraction_search'`, `freeOnly: true`. (depends on M22-04)
   - `"cheap salsa in Laureles"` → `intent: 'event_discovery'`, category includes `nightlife|salsa`.
   - `"coffee in Poblado"` → `intent: 'restaurant_search'`, `neighborhood: 'El Poblado'`.
   - `"hi"` → `intent: 'chitchat'`.
   - `"asdf"` → `intent: 'unknown'`.
4. Wire `npm test` into the smoke flow so CI runs it before `mastra build`.
