## Annotations

| Element | Component | Interaction |
|---------|-----------|-------------|
| Search bar | `<Input>` + `<Button>` | Text search filters results in real-time via debounce |
| Neighborhood checkboxes | `<Checkbox>` from shadcn | Multi-select, updates URL params, triggers TanStack Query refetch |
| Price range slider | `<Slider>` (dual thumb) | Min/max range, shows values above thumbs |
| Bedrooms select | `<Select>` | Options: Any, Studio, 1, 2, 3, 4+ |
| Furnished toggle | `<RadioGroup>` | Yes / No / Any |
| Amenities multi-select | `<Checkbox>` group | Each toggles a filter tag |
| Wi-Fi speed | `<Select>` | Options: Any, 20+, 50+, 100+ Mbps |
| Move-in date | `<Calendar>` via `<Popover>` | Date picker, filters to available-by-date |
| Freshness badge | Custom `<Badge>` | Green=<3d, Yellow=<1w, Gray=>1w. Uses listing `created_at` |
| Result cards | `<Card>` | Hover: subtle shadow. Click: populates right panel detail |
| Schedule Showing button | `<Button variant="default">` | Opens showing scheduler dialog (see wireframe 05) |
| Map pins | Google Maps + custom overlays | Price label pins. Click pin = select card. Hover = highlight |
| Sort dropdown | `<Select>` | Options: Newest, Price Low-High, Price High-Low, Best Match |
| Pagination | `<Pagination>` | Server-side, 12 results per page |
| FloatingChatWidget | Fixed position bottom-right | Opens AI chat panel. Persists across pages |
| Clear/Apply filters | `<Button variant="outline">` / `<Button>` | Clear resets all, Apply triggers search |

## Data Flow

1. Filters update URL search params via `useSearchParams`
2. `useApartments` hook reads params, calls Supabase `.from('apartments').select(...)` with filters
3. TanStack Query caches results, shows stale-while-revalidate
4. Map syncs with visible results (bounds filtering optional)
5. Card click sets `selectedId` state, right panel fetches detail via `useApartment(id)`
