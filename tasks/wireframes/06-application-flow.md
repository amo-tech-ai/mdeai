## Annotations

| Element | Component | Interaction |
|---------|-----------|-------------|
| Progress stepper | Custom component with `●`/`○` + connecting lines | Filled circles for completed steps, current step highlighted |
| Form fields | `<Input>`, `<Select>`, `<RadioGroup>` from shadcn | All validated with Zod schemas via react-hook-form |
| Date picker | `<Calendar>` in `<Popover>` | Standard shadcn date picker pattern |
| File upload | Custom drag-and-drop zone | Accepts PNG, JPG, PDF. Max 5MB. Shows progress bar |
| Reference inputs | Dynamic field array | "Add another" appends new reference row |
| Review sections | `<Card>` with [Edit] buttons | Edit navigates back to that step, preserving other data |
| Checkboxes | `<Checkbox>` | Both required for Submit to be enabled |
| Submit button | `<Button variant="default">` | Disabled until both checkboxes checked + all required fields |
| Timeline | Custom vertical stepper | Visual timeline of next steps after submission |
| Back/Next | `<Button variant="outline">` / `<Button>` | Navigate between steps, data preserved in form state |

## Form Validation (Zod)

```
Step 1: name (required), email (valid email), phone (required), nationality (required), occupation (required)
Step 2: moveInDate (future date), duration (enum), occupants (1-10), pets (enum), purpose (enum)
Step 3: govId (file required), proofOfIncome (file required), references (optional array)
Step 4: termsAccepted (true), infoAccurate (true)
```

## Backend

- Application stored in `rental_applications` table
- Files uploaded to Supabase Storage (`applications/{user_id}/{file_name}`)
- Status flow: `submitted` -> `under_review` -> `approved` / `rejected`
- Notification sent to host on submit (in-app + email)
- Notification sent to renter on status change
