## 2 · Hidden bugs found

I dug for these specifically since the user said "do not assume it works."

|Severity|Bug|Mitigation status|
|---|---|---|
|🟡|**Telemetry sink default is `console.debug`** — most browsers hide debug-level logs; you have to enable "Verbose" in DevTools to see them. Could create false confidence that nothing is firing.|Not a code bug; telemetry sink is pluggable. **Fix:** wire Sentry/PostHog (next-step).|
|🟡|`viewport_idle` event **type** is defined in `MapTelemetryEvent` but **no call site emits it**. Type allows it; reality doesn't fire it yet.|Documented in audit as "wired in 60-day plan." Honest.|
|🟡|**`onClusterClick` uses `cluster.markers?.length`** — `cluster.count` is the documented field. `markers` is also there but is a Marker[] internal collection. Both work; `count` is more stable.|One-char change in next refactor.|
|🟢|When user signs in mid-session **after** sending an anon message, `clearPins` does NOT fire (auth state change ≠ conversation change). Their anon-search pins persist into their first authed conversation.|Minor. The new authed conversation is a separate row, so subsequent searches in that conversation will set fresh pins; the brief overlap is harmless.|
|🟢|The `clearPins` useEffect fires **on every mount** because `currentConversation?.id` is undefined → undefined comparison still triggers a no-op call.|No-op, no perf cost. Could add `if (!currentConversation) return;` for cleanliness.|
|🟢|A11y: our custom marker `<div role="button">` is a child of Maps' `<gmp-advanced-marker>` which is itself focusable. Screen readers may announce both.|NVDA/JAWS quirk; minor. Real fix is removing `role="button"` since the parent already has button semantics — but the `aria-label` MUST stay.|

**No critical bugs found. Three real-but-tolerable yellows. Three minor greens.**

## 3 · Console errors / red flags / runtime issues

In the production bundle, every error path leads to `trackMapEvent({ kind: '*_failed' })` + `console.error` (or `setMapError` for UI-recoverable cases). I couldn't find an unhandled throw path in the maps stack.

**Red flags (architectural, not bugs):**

- 🟡 **Bundle size: 1.81 MB / ~480 KB gzip.** MarkerClusterer added ~12 KB. No code-splitting yet. LATAM 4G users → 2-4s first paint penalty.
- 🟡 **No real telemetry pipe.** Console-only sink means the events fire but nobody is watching. You'll be blind to prod issues until Sentry is wired.
- 🟡 **MapProvider is chat-only.** Apartment detail / trips pages don't share pin state. Cross-page coherence is a 60-day item.
- 🟡 **No clusterer renderer customization.** Default Google blue circle clusters look generic. Mindtrip styles theirs.

**No memory leaks.** Verified via the unmount cleanup effect that removes both `gmp-click` listeners and clears the clusterer.
