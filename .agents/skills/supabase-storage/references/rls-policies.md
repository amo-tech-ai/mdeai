# Supabase Storage RLS Policy Reference

All policies target `storage.objects`. Always use `(SELECT auth.uid())` not `auth.uid()` directly.

## Public bucket — read-only (anyone can read)

```sql
CREATE POLICY "public_read"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'avatars');
```

## Authenticated users can read all files in a bucket

```sql
CREATE POLICY "auth_read"
  ON storage.objects FOR SELECT TO authenticated
  USING (bucket_id = 'documents');
```

## User-scoped files (user_id as first path segment)

```sql
-- Allow user to upload their own files: avatars/<user_id>/filename
CREATE POLICY "user_upload"
  ON storage.objects FOR INSERT TO authenticated
  WITH CHECK (
    bucket_id = 'avatars'
    AND (storage.foldername(name))[1] = (SELECT auth.uid()::text)
  );

CREATE POLICY "user_update"
  ON storage.objects FOR UPDATE TO authenticated
  USING (
    bucket_id = 'avatars'
    AND (storage.foldername(name))[1] = (SELECT auth.uid()::text)
  );

CREATE POLICY "user_delete"
  ON storage.objects FOR DELETE TO authenticated
  USING (
    bucket_id = 'avatars'
    AND (storage.foldername(name))[1] = (SELECT auth.uid()::text)
  );
```

## Service-role-only bucket (edge functions write; users read via signed URL)

```sql
-- No INSERT/UPDATE/DELETE policies at all → only service_role can write
-- Users read via signed URLs generated by edge functions

-- If you want authenticated users to read their own signed-URL-accessed files:
CREATE POLICY "owner_read"
  ON storage.objects FOR SELECT TO authenticated
  USING (
    bucket_id = 'contracts'
    AND owner = (SELECT auth.uid())
  );
-- 'owner' is populated automatically to auth.uid() when a user uploads
-- For service_role uploads, set owner explicitly or leave null (edge fn pattern)
```

## Sponsor contracts — org-owner reads, service_role writes

```sql
-- Sponsor can read their own contract files
-- Path convention: contracts/<application_id>/contract-v1.html
CREATE POLICY "sponsor_read_contracts"
  ON storage.objects FOR SELECT TO authenticated
  USING (
    bucket_id = 'contracts'
    AND (storage.foldername(name))[1] IN (
      SELECT a.id::text
      FROM sponsor.applications a
      JOIN sponsor.organizations o ON o.id = a.organization_id
      WHERE o.primary_contact_user_id = (SELECT auth.uid())
    )
  );
-- No INSERT/UPDATE/DELETE policies → service_role edge fn only
```

## Organizer reads event media, anyone can read public media

```sql
-- Event media: events/<event_id>/banner.jpg
CREATE POLICY "organizer_write_event_media"
  ON storage.objects FOR INSERT TO authenticated
  WITH CHECK (
    bucket_id = 'event-media'
    AND (storage.foldername(name))[1] IN (
      SELECT id::text FROM public.events
      WHERE organizer_id = (SELECT auth.uid())
    )
  );

CREATE POLICY "public_read_event_media"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'event-media');
```

## Admin-only bucket

```sql
CREATE POLICY "admin_only"
  ON storage.objects FOR ALL TO authenticated
  USING (
    bucket_id = 'admin-docs'
    AND EXISTS (
      SELECT 1 FROM public.profiles
      WHERE id = (SELECT auth.uid())
      AND role IN ('admin', 'super_admin')
    )
  )
  WITH CHECK (
    bucket_id = 'admin-docs'
    AND EXISTS (
      SELECT 1 FROM public.profiles
      WHERE id = (SELECT auth.uid())
      AND role IN ('admin', 'super_admin')
    )
  );
```

## Tips

- `storage.foldername(name)` splits the path on `/` and returns an array. Index 1-based.
- `storage.filename(name)` returns the last segment (filename with extension).
- `storage.extension(name)` returns just the extension (e.g. `'pdf'`).
- Add a GIN or B-tree index on `(bucket_id, name)` in `storage.objects` if RLS policies do prefix scans.
- Use `EXPLAIN ANALYZE` on RLS-protected queries to detect sequential scans on large buckets.
